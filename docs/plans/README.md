# Implementation Plans

This directory contains detailed technical specifications and implementation plans for features.

## When to Write a Plan

- Feature requires changes across multiple layers (frontend, backend, database)
- Implementation approach isn't obvious
- You want to think through edge cases before coding
- Coordinating work that spans multiple PRs

## Template

```markdown
# Feature Name

**Issue:** #123
**Date:** YYYY-MM-DD
**Status:** Draft | In Progress | Completed | Abandoned

## Overview

Brief description of what we're building and why.

## Requirements

- [ ] Requirement 1
- [ ] Requirement 2

## Technical Approach

### Backend Changes
- ...

### Frontend Changes
- ...

### Database Changes
- ...

## Open Questions

- Question 1?
- Question 2?

## Tasks

- [ ] Task 1
- [ ] Task 2
```
